# HTML 和静态资源

# HTML

# Index 文件

public/index.html 文件是一个会被 html-webpack-plugin (opens new window) 处理的模板。在构建过程中，资源链接会被自动注入。

# 插值

因为 index 文件被用作模板，所以你可以使用 lodash template (opens new window) 语法插入内容：

  • <%= VALUE %> 用来做不转义插值；
  • <%- VALUE %> 用来做 HTML 转义插值；
  • <% expression %> 用来描述 JavaScript 流程控制。

所有定义配置文件var的变量都可以使用

<link rel="icon" href="<%= BASE_URL %>favicon.ico" />

# 处理静态资源

放置在 public 目录下的这类资源将会直接被拷贝，而不会经过 webpack 的处理

不过public文件夹存放资源并不推荐，因为使用webpack构建会使资源文件的排列更加规则而且还可以对资源进行特定的处理

# 何时使用 public 文件夹

  • 你需要在构建输出中指定一个文件的名字。
  • 你有上千个图片，需要动态引用它们的路径。
  • 有些库可能和 webpack 不兼容，这时你除了将其用一个独立的 <script> 标签引入没有别的选择。
